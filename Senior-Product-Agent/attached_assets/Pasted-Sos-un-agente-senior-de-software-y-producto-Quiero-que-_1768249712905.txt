Sos un agente senior de software y producto. Quiero que crees una app web MUY SIMPLE (MVP) para llevar el registro de mis aportes a un proyecto inmobiliario. Tiene que correr en Replit FREE y deployar en Vercel FREE sin costos.

OBJETIVO
- App mínima, rápida, sin dependencias raras, sin DB externa.
- Persistencia local con archivo JSON (para no usar Supabase/Firebase).
- UI simple para cargar aportes y ver totales.
- Preparada para luego subir a GitHub y deployar en Vercel.

STACK OBLIGATORIO
- Next.js (App Router) + TypeScript
- Sin base de datos externa.
- Persistencia: archivo local JSON usando el filesystem en el server (API routes).
- Estilos: CSS simple (sin Tailwind, sin UI kits).
- Evitar heavy tooling. Nada de Prisma, Supabase, Auth, etc.

FUNCIONALIDADES MVP
1) Página principal “Dashboard”
   - Título: “Mi Inversión”
   - Resumen con:
     - Total aportado en USD
     - Total aportado en ARS
     - Lista de movimientos (ordenados por fecha desc)

2) Movimientos (CRUD básico)
   - Un movimiento tiene:
     - id (uuid)
     - date (YYYY-MM-DD)
     - type: "APORTE" | "COBRO"
     - currency: "USD" | "ARS"
     - amount (number)
     - note (string opcional)
     - fxType (solo si currency = ARS): "BLUE" | "OFICIAL" | "MANUAL"
     - fxRate (solo si fxType = MANUAL): number
     - usdEquivalent (calculado y guardado si currency=ARS)
   - Formularios:
     - Agregar movimiento
     - Eliminar movimiento
     - (Opcional si es rápido) Editar movimiento

3) Tipo de cambio (simple y robusto)
   - Si currency=ARS:
     - Permitir elegir fxType: BLUE / OFICIAL / MANUAL
     - Si BLUE u OFICIAL:
       - Traer tipo de cambio desde una API pública (la que elijas) con fetch server-side.
       - Guardar el valor usado en el movimiento (para que no cambie en el tiempo).
     - Si MANUAL:
       - Usuario escribe fxRate (ARS por USD)
   - Mostrar USD equivalente = amount_ars / fxRate

4) Comparación con SP500 (simple, sin data compleja)
   - Permitir setear un “Benchmark anual %” editable (por default 10%).
   - Calcular “valor hipotético hoy” por cada aporte USD:
     - Usar interés compuesto desde la fecha del aporte hasta hoy:
       value = aporteUSD * (1 + r)^(días/365)
   - Mostrar:
     - Total invertido USD
     - Valor hipotético S&P (aprox) USD
     - Diferencia

IMPORTANTE (para no romper en Vercel)

- Aun así, quiero que el proyecto compile y funcione en Vercel:
  - Implementar una capa de storage con interfaz:
    - storage/fileStorage.ts (default en Replit)
    - storage/memoryStorage.ts (fallback si FS no es escribible)
  - Detectar si se puede escribir en /tmp (Vercel) y usar /tmp/data.json si es posible.
  - Si no, usar memoryStorage y avisar en UI: “Modo demo: datos no persistentes”.
